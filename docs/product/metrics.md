## Success Criteria

### Week 1

- Open AIS and SAR data sources are selected and fixed.
- Request queries are reviewed, and response parameters are understood and documented.
- Known challenges and data limitations are identified and described at a high level.
- Basic user workflows are defined and agreed.
- A project Git repository is created.
- A project name is defined.
- A minimal application is created with initial required packages (e.g., map display and state management).

## Month 1

- A basic API is available to handle communication with data sources and datasets.
- A simple UI is available with basic filter inputs (drawing an area of interest and selecting time ranges).
- Heatmap and report data can be requested and retrieved.
- Detected vessels can be displayed on the map.
- A list view of vessel detections is available.
- Available AIS messages can be retrieved for a specific vessel MMSI.
- Automated tests are defined, including unit tests and basic end-to-end tests.
- Known data gaps and limitations are documented and visible to the user or reviewer.
